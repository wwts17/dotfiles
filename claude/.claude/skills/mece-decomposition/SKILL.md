# MECE 分解

## 核心
把问题拆成若干子问题,两两不重叠、合起来无遗漏。

## 适用场景
拆需求、切项目、设计测试矩阵、列方案选项,问题边界模糊时。
不用:问题已经定义清楚,再拆是过度设计。

## 思考步骤
1. 用一句话写下问题
2. 列出候选子问题
3. 查相互独立:任意两个之间无重叠
4. 查完全穷尽:每种情况(含边界 case)都能归入某一类
5. 迭代到两条都满足,且各分支抽象层级一致

## 输出结构
- 问题:提升注册转化率
- 子问题(MECE):
  - A 减少表单摩擦
  - B 增强信任信号
  - C 提供替代路径(社交登录)
- 检验:A/B/C 互不重叠;任何改进都属于"结构/感知/替代"之一,无遗漏
