# Git 提交

## 核心
提交信息让人一眼看懂改了什么、为什么;遵循 Conventional Commits,禁任何 AI 署名。

## 适用场景
创建 git 提交或撰写提交信息时。

## 规则
格式:`<type>(<scope>): <主题>`,空行后接可选正文/脚注。

| type | 含义 | | type | 含义 |
|---|---|---|---|---|
| feat | 新功能 | | refactor | 重构(不改行为) |
| fix | 修 bug | | test | 测试 |
| docs | 文档 | | chore | 构建/工具 |
| style | 格式(不改逻辑) | | perf | 性能 |

- 主题:祈使句、小写、不加句号、≤50 字
- 正文:解释**为什么**,不解释做了什么;每行 ≤72 字
- 破坏性变更脚注 `BREAKING CHANGE: <说明>`;关联需求脚注 `Refs: <ID>`
- **不加任何 AI 署名**(`Co-Authored-By`、`Generated by` 等);作者就是 git 配置的身份

## 范例
❌ `git commit -m "update"`——看不出改了什么
✅
```
fix(cart): correct tax calculation for EU orders

EU 订单按目的地国税率计税,之前误用了卖家所在地税率。
```
